# Definition of Done

## 1. Purpose

Зафиксировать критерии завершённости по проекту в целом и по областям: страницы, CMS, формы, блог/статья, SEO, страницы услуг, UI и адаптивность. Используется для приёмки и QA (в т.ч. Phase 8).

## 2. Scope

Глобальный DoD и пошаговые DoD по каждой области. Чеклист для проверки перед сдачей этапа или всего проекта.

---

## 3. Global DoD (проект в целом)

- [ ] Все публичные страницы из ТЗ реализованы и доступны по заложенным маршрутам (03).
- [ ] Админка Payload доступна по /admin; авторизация через env; все коллекции работают (08).
- [ ] Контент главной и страниц услуг соответствует ТЗ (05, 06, 15); контент статей в CMS (07, 08).
- [ ] Формы обратной связи и подписки сохраняют данные и отправляют письма; данные видны в админке (10).
- [ ] SEO: метаданные, canonical, OG, Twitter, structured data (Organization, BlogPosting), sitemap.xml, robots.txt (11).
- [ ] Медиа загружаются в public/uploads с валидацией; обложки и изображения в статьях отображаются (12).
- [ ] GTM подключается через env (13).
- [ ] 404 и privacy policy в общем дизайне; документация (README, env, контент) актуальна (19, 16 Phase 8).
- [ ] Нет критичных багов по результатам QA; известные ограничения зафиксированы (17).

---

## 4. DoD для страниц (общее)

- [ ] Страница отображается без ошибок в консоли и без падения при навигации.
- [ ] Метаданные (title, description) заданы и релевантны странице (11).
- [ ] Canonical URL корректен (11).
- [ ] Адаптивная вёрстка в пределах заданных breakpoints; контент читаем и не обрезан (04).
- [ ] Ссылки и кнопки ведут на правильные цели; якоря работают (при наличии).
- [ ] Header и Footer едины по сайту (если по дизайну).

---

## 5. DoD для CMS

- [ ] Коллекции Articles, Categories, Tags, Media, Leads, NewsletterSubscriptions созданы с полями по 08.
- [ ] Статьи: создание, редактирование, переключение draft/published; slug уникален; rich text с заголовками, цитатами, ссылками, изображениями, alt (08).
- [ ] Категории и теги: CRUD; slug уникален (08).
- [ ] Медиа: загрузка с валидацией типа и размера; файлы в public/uploads (08, 12).
- [ ] Заявки и подписки: только просмотр в админке; данные создаются через формы (08, 10).
- [ ] Миграции применяются; seed создаёт тестовые данные (14, 09).

---

## 6. DoD для форм

- [ ] Форма обратной связи: поля name, email, message (и при необходимости phone); серверная валидация (10).
- [ ] После отправки заявка сохраняется в Leads; отображается в админке (10).
- [ ] Пользователю и администратору отправляются письма после отправки формы (10).
- [ ] Форма подписки: поле email; валидация и проверка уникальности (10).
- [ ] Подписка сохраняется в NewsletterSubscriptions; письма пользователю и админу отправляются (10).
- [ ] На фронте отображается успех или ошибка; при ошибке валидации показываются поля с ошибками (10).
- [ ] SMTP настраивается через env; HTML-шаблоны писем используются (10).

---

## 7. DoD для блога и статей

- [ ] Архив /blog: список только published статей; пагинация; ссылки на категории и на статью (07).
- [ ] Страница категории /blog/c/[category]: название, список статей категории, пагинация; 404 при неизвестном slug (07).
- [ ] Страница статьи /blog/[slug]: breadcrumbs, заголовок, дата, автор 8Blocks, время чтения (вычисляемое), обложка, rich text, ToC с якорями (07).
- [ ] Rich text: заголовки (с id), параграфы, цитаты, ссылки, изображения с alt (07).
- [ ] ToC строится автоматически; клик по пункту скроллит к заголовку (07).
- [ ] Просмотры: учёт через API, не чаще одного раза за сессию (07).
- [ ] Draft не отображается на фронте; при noindex в метаданных страницы задан noindex (07, 11).
- [ ] Метаданные и structured data BlogPosting для статьи (11).

---

## 8. DoD для SEO

- [ ] У всех публичных страниц заданы title, description, alternates.canonical (11).
- [ ] OpenGraph и Twitter card заполнены (title, description, image, url) (11).
- [ ] Для статей: type article, publishedTime; при noindex — robots noindex (11).
- [ ] JSON-LD Organization на главной; BlogPosting на статье (11).
- [ ] sitemap.xml генерируется динамически; включены статика, категории, опубликованные статьи; noindex-статьи не включены (11).
- [ ] robots.txt: Disallow /admin, /api; Sitemap указан (11).
- [ ] SITE_URL используется для canonical и sitemap; задан в env в production (11).

---

## 9. DoD для страниц услуг

- [ ] Страница всех услуг: заголовок, описание, три карточки со ссылками на лендинги (06).
- [ ] Strategic Consulting: все секции (Hero, Problem, Solution, Deliverables, CTA, Process, Use cases, FAQ); 11 пунктов FAQ в аккордеоне (06).
- [ ] Tokenomics: все секции включая Mini App/Demo (placeholder допустим) и 11 пунктов FAQ (06).
- [ ] Audit: все секции; место под FAQ; при необходимости минимальный контент для What you get и False assumptions (06, 15).
- [ ] CTA на всех страницах ведут к форме обратной связи (#contact или аналог) (06).
- [ ] Визуально страницы не шаблонные; единый дизайн по 04 (06).

---

## 10. DoD для главной страницы

- [ ] Секции: Hero, Services, About, Partners, Benefits, CTA; Footer с формой (05).
- [ ] Все тексты из ТЗ размещены в соответствующих блоках (05, 15).
- [ ] Якоря: Hero → #services; CTA → скролл к форме в footer (05).
- [ ] Карточки услуг ведут на /services/strategic-consulting, /services/tokenomics, /services/audit (05).
- [ ] Секция услуг адаптивна; партнёры — карусель или сетка по дизайну (05).
- [ ] Structured data Organization на главной (11).

---

## 11. DoD для responsive and UI polish

- [ ] Design tokens из ТЗ применены (цвета, шрифт Manrope, границы, акценты) (04).
- [ ] Mantine используется только из разрешённого списка; основной UI кастомный через SCSS (04).
- [ ] Нет горизонтального скролла на мобильных; контент читаем и кнопки доступны (04).
- [ ] Контраст текста и кнопок проверен (доступность); фокус виден при навигации с клавиатуры (04).
- [ ] Анимации (Framer Motion) и Lenis не ломают навигацию и скролл (04, 05).

---

## 12. DoD для privacy policy и 404

- [ ] Страница /privacy-policy содержит все блоки из ТЗ (вступление, определения, сбор данных, дети, ссылки, изменения, контакты) (19, 15).
- [ ] Дата обновления и контакт (hello@8blocks.io) указаны (19).
- [ ] Метаданные страницы заданы (11).
- [ ] 404 (not-found): отображается в общем дизайне сайта; noindex (11).

---

## 13. Использование

- На этапе Phase 8 пройти по всем пунктам DoD и отметить выполненное.
- Невыполненные пункты — в бэклог или в scope до сдачи.
- При приёмке финального результата использовать глобальный DoD и области по затронутым частям.
