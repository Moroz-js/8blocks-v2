# Design System and UI Rules

## 1. Purpose

Зафиксировать дизайн-токены, правила использования Mantine, организацию SCSS и типографики. Обеспечить единообразие UI по всему проекту.

## 2. Scope

- Цвета, типографика, границы (из ТЗ).
- Ограничения по Mantine: перечень компонентов и правило «не задаёт общий стиль».
- Организация global.scss и модулей.
- Базовые UI-компоненты и их использование на страницах.

## 3. Inputs from requirements

- Цвета фона: background-primary `#000000`, background-secondary `#0B0B0B`, background-tertiary `#29292A`.
- Текст: text-primary `#FFFFFF`, text-secondary `rgba(255,255,255,0.5)`, text-tertiary `rgba(255,255,255,0.4)`.
- Границы: border-primary `rgba(255,255,255,0.2)`, border-secondary `rgba(255,255,255,0.07)`.
- Акценты: purple `#C53DFF`, green `#75FB63`, blue `#638EFB`.
- Шрифт: Manrope (Regular, Medium).
- Mantine: только перечисленные компоненты; основной UI — кастом через SCSS и проектные компоненты.
- Framer Motion — анимации; Lenis — плавный скролл; clsx — классы; Swiper/Embla только при необходимости.

## 4. Design tokens (реализация)

### 4.1 CSS-переменные (рекомендуемый способ)

В `globals.scss` (или отдельном файле токенов) задать переменные для использования в модулях и переопределения Mantine при необходимости:

```scss
// Пример структуры — не production code, только планирование
:root {
  --background-primary: #000000;
  --background-secondary: #0B0B0B;
  --background-tertiary: #29292A;
  --text-primary: #FFFFFF;
  --text-secondary: rgba(255, 255, 255, 0.5);
  --text-tertiary: rgba(255, 255, 255, 0.4);
  --border-primary: rgba(255, 255, 255, 0.2);
  --border-secondary: rgba(255, 255, 255, 0.07);
  --accent-purple: #C53DFF;
  --accent-green: #75FB63;
  --accent-blue: #638EFB;
  --font-manrope: 'Manrope', sans-serif;
}
```

### 4.2 Типографика

- Подключение Manrope (Google Fonts или локальные файлы) в layout или globals.
- Классы или миксины для заголовков и тела текста (Regular / Medium).
- Размеры и отступы — зафиксировать в токенах или в отдельном типографическом файле (например h1–h4, body, small, caption).

## 5. Mantine: разрешённые компоненты и правила

**Разрешено использовать только**:

- Button, TextInput, Textarea, Select, Menu, Modal, Accordion, Tabs, Checkbox, Radio, Switch, Drawer, Notification.

**Правила**:

- Mantine не определяет общий визуальный стиль: цвета, скругления, тени должны задаваться проектной темой или переопределением через CSS/SCSS.
- Использовать Mantine для логики (аккордеон, модалка, табы), а внешний вид доводить кастомными классами (например, Accordion для FAQ с проектной раскраской и типографикой).
- Не использовать компоненты Mantine, не входящие в список (например Card, Stack, Group — реализовать свои в shared/ui или виджетах на базе div + SCSS).

## 6. SCSS: организация

- **globals.scss**: сброс, токены, шрифты, базовые утилиты (при необходимости).
- **Модули**: один `.module.scss` на компонент/виджет; классы по BEM или по конвенции проекта; импорт в тот же компонент.
- **Общие миксины/переменные**: вынести в `shared/styles/` (например `_variables.scss`, `_mixins.scss`) и импортировать в модули при необходимости.
- Имена классов: осмысленные, без конфликтов (модули изолируют по имени файла).

## 7. Базовые UI-компоненты (shared/ui)

Перечень компонентов, которые стоит выделить для переиспользования:

- **Button** — первичный, вторичный, с иконкой (если в дизайне есть); стили из токенов.
- **Card** (или SectionCard) — для карточек услуг, benefits, статей; фон, граница, отступы из токенов.
- **Input / Textarea** — обёртки над Mantine TextInput/Textarea с проектной стилизацией.
- **Accordion** — обёртка над Mantine Accordion для FAQ с единым стилем.
- **Link** — стилизованные ссылки (в т.ч. якорные).
- **Container / Section** — обёртки для секций (max-width, padding) для единообразия отступов.
- **Icon** — если используются иконки (SVG sprite или компоненты); единый размер/цвет по контексту.

Если в проекте уже есть кнопки с иконками (из правил пользователя — «existing styles»), все новые кнопки с иконками должны использовать тот же паттерн.

## 8. Анимации и скролл

- **Framer Motion**: для появления секций, hover-эффектов, модалок — по необходимости; не перегружать главную страницу.
- **Lenis**: плавный скролл подключается на уровне layout; учесть совместимость с якорными ссылками (Hero → Services).
- **Swiper/Embla**: только если в дизайне явно нужен слайдер (например партнёры); иначе — CSS grid/flex.

## 9. Responsive

- Breakpoints задать единообразно (например в переменных или в конфиге): mobile, tablet, desktop.
- Секции главной и страниц услуг должны быть адаптивными (указано в ТЗ); карточки услуг — адаптивная сетка.
- Контейнеры и отступы масштабировать по breakpoints.

## 10. Dependencies

- Токены и globals должны быть готовы в Phase 0–1 до массовой вёрстки виджетов.
- Mantine и остальные зависимости установлены в Phase 0.

## 11. Risks / open questions

- Tailwind: в user rules указан Tailwind; в ТЗ — SCSS Modules. Для planning принимаем ТЗ (SCSS). Если проект позже перейдёт на Tailwind, токены нужно будет продублировать в tailwind.config.
- Контраст акцентных цветов на тёмном фоне — проверить на доступность (WCAG).

## 12. Deliverables

- Файл(ы) с design tokens (CSS variables или SCSS variables).
- globals.scss с подключением шрифтов и токенов.
- Общие компоненты в shared/ui с проектной стилизацией.
- Документ или комментарии с перечнем разрешённых Mantine-компонентов.

## 13. Acceptance criteria

- Все цвета и шрифты из ТЗ доступны через токены.
- Mantine используется только из разрешённого списка и не задаёт глобальный визуальный стиль.
- Новый виджет использует те же токены и общие компоненты; нет дублирования цветов/шрифтов в коде.
